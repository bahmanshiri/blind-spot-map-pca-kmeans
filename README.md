# Mapping the Blind Spots of PCA+K-means

Reproducibility package for:

> Shiri Lonbar B, Hassanifar Z. "Mapping the Blind Spots of PCA+K-means:
> A Diagnostic Framework and Geometric Criterion for Minority Risk Segment
> Detection."

This repository contains all code, intermediate results, and figures
needed to reproduce the experiments reported in the paper, organized by
manuscript section.

## Repository structure

```
code/
  section3_primary_dataset/    Motivating case study (synthetic banking-analogue data)
  section4_synthetic_grid/     9x9 sensitivity grid + causal decomposition (2,025 runs)
  section5.4_dry_bean/         Real-data validation on UCI Dry Bean
  section5.5_ieee_cis/         Real-data validation on IEEE-CIS fraud transactions
data/
  dry_bean_clean.csv           Cleaned Dry Bean dataset (public, UCI)
  primary_dataset_clean.csv    Generated synthetic banking-analogue dataset
  raw/                         Where to place the IEEE-CIS raw file (see below)
results/
  section5.4_dry_bean/         Result CSVs (ARI grids, algorithm comparison, criterion validation)
  section5.5_ieee_cis/         Result CSVs (ratio sweep, PCA-dimension sweep, noise robustness, criterion validation)
section3/                      Figures for the motivating case study (Section 3)
section4/                      Figures for the synthetic sensitivity grid (Section 4)
section5.4_dry_bean/           Figures for the Dry Bean validation (Section 5.4)
section5.5_ieee_cis/           Figures for the IEEE-CIS validation (Section 5.5)
requirements.txt
```

Note: the four top-level folders `section3/`, `section4/`,
`section5.4_dry_bean/`, and `section5.5_ieee_cis/` contain only the
**figures** for each section (PNG/TIFF), matching the paper's Fig. N
numbering. They are separate from the identically-named subfolders
under `code/`, which contain the analysis **scripts** for those same
sections.

## Setup

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Python 3.10+ recommended.

## Reproducing the results

Each section's scripts are numbered in the order they should be run.
Run them from inside each section's `code/` folder (paths are relative
to the script location).

- **Section 3** (motivating case study): run `01_generate_dataset.py`,
  then `02_primary_pipeline.py`, then `03_figures.py`.
- **Section 4** (synthetic 9x9 grid, 2,025 runs): run `06_blindspot_grid.py`,
  then `07_dilution_confirmation.py`, then `08_figures.py`.
  `00_grid_data_utils.py` is a shared helper module, not run directly.
- **Section 5.4** (Dry Bean): the cleaned dataset is already included at
  `data/dry_bean_clean.csv`, generated from the public UCI Dry Bean
  Dataset. Run `01_load_and_clean.py` through `06_theory_grid_validation.py`
  in numeric order.
- **Section 5.5** (IEEE-CIS fraud): the raw Kaggle file is **not** included
  in this repository (see `data/raw/README.md` for how to obtain it).
  Once `train_transaction.csv` is in place, run `01_load_and_clean.py`
  through `06_geometric_criterion_ieee.py` in numeric order.
  `data_utils.py` is a shared helper module, not run directly.

All random seeds are fixed in the scripts; results should match the CSVs
already provided under `results/` and the numbers reported in the paper.

## Data sources

- **UCI Dry Bean Dataset**: Koklu M, Ozkan IA (2020). Multiclass
  classification of dry beans using computer vision and machine learning
  techniques. *Computers and Electronics in Agriculture*, 174, 105507.
  https://doi.org/10.24432/C50S4B
- **IEEE-CIS Fraud Detection**: Kaggle competition dataset, subject to
  competition-specific redistribution terms. See `data/raw/README.md`.
- **Synthetic banking-analogue data**: generated by
  `code/section3_primary_dataset/01_generate_dataset.py` and
  `code/section4_synthetic_grid/00_grid_data_utils.py`; no external
  source required.
